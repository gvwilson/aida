## Analysis

The rstx chapters contain systematic, quantitative verification and validation advice
absent from the aida lessons. Key gaps by category:

### Tolerance justification

rstx chapters explain *why* a particular numeric tolerance was chosen (e.g., "5×10^-3
is 1.7× the measured error," "tolerance gives 7.5× safety factor"). aida recommends
spot-checking LLM output but never discusses how to decide how close is close enough.

### Edge cases and boundary testing

rstx chapters test boundaries explicitly: identical inputs, zero inputs, single-row
grids, thresholds at/above maximum values. aida recommends checking results look
reasonable but not systematic boundary design.

### Invariant and conservation checking

rstx chapters verify quantities that must be preserved: mass conservation (pollute,
diffusion), population totals (sir, wealth, schelling), probability axioms (sdt, flood).
aida has no equivalent emphasis on invariants as a testing strategy.

### Analytical / closed-form comparison

rstx chapters validate against known exact solutions when available (noise-free
parameter recovery, Gaussian diffusion profiles, algebraic identities). aida does not
recommend deriving special cases with computable exact answers.

### Alternative method comparison

rstx chapters compare results across methods (Pearson III vs. log-normal for flood
return levels; linearized vs. measured period in Lotka-Volterra). aida does not
recommend cross-method validation.

### Reproducibility via random seeds

rstx chapters test that calling a function twice with the same seed returns identical
arrays. aida mentions versioning but not seed-based reproducibility as a verification
technique.

### Sensitivity and finite-size analysis

rstx chapters vary parameters (grid size, noise scale) and track how errors change.
aida has no equivalent guidance on parameter sweeps to confirm robustness.

### Synthetic data calibration

rstx chapters design synthetic test data at a known signal-to-noise ratio so that
test difficulty is calibrated, not arbitrary. aida generates synthetic data without
this calibration step.

### Hand-calculation checks

rstx chapters recommend tracing through a small example by hand (e.g., sdt ROC curve
for a handful of scores) to catch implementation bugs. aida does not suggest this.

### Directional bias detection

rstx chapters check for known directional artifacts (e.g., naive survival estimate
always exceeds corrected estimate when censoring exists). aida recommends comparison
but not asymmetry analysis.
